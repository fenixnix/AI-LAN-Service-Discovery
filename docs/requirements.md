# AIEcho 文档

## 系统 - 需求1. 项目概述

### 1.1 项目名称

**AIEcho**

### 1.2 项目目标

设计并实现一套轻量级、零配置、高性能的局域网 AI 微服务发现机制，使 AI Agent 能够动态发现并调用局域网内部署的各种 AI 工具服务（如 PDF 处理、图像生成、知识库检索等）。

核心目标：
1. **毫秒级发现** - UDP 广播机制，10ms 内发现局域网内所有可用服务
2. **零代码接入** - 仅需 JSON 配置文件即可完成服务注册，无需修改业务代码
3. **实时感知** - 服务上线/下线自动公告，客户端动态更新服务列表
4. **多语言支持** - 提供 Python 和 Rust 原生实现库

### 1.3 目标用户

- **AI 开发者**: 需要在局域网内动态发现和调用各种 AI 工具的开发者
- **运维人员**: 需要管理局域网内多个 AI 服务的运维人员
- **AI Agent**: 需要自主发现可用工具的 AI 代理程序

## 2. 功能需求

## 3. 非功能需求

### 3.1 性能要求

- **发现延迟**: UDP 广播发现延迟 < 10ms
- **Manifest 获取**: 单个服务 Manifest 获取 < 50ms
- **并发支持**: 支持 100+ 服务同时在线
- **内存占用**: 单次扫描内存 < 1MB
- **CPU 占用**: 持续运行 CPU < 1%

### 3.2 可用性要求

- **系统可用性**: 99.9%+
- **启动时间**: 服务代理启动 < 2 秒
- **故障恢复**: 服务异常退出后，客户端能在 30 秒内感知

### 3.3 安全要求

- **网络隔离**: 仅限局域网内通信，不暴露到公网
- **防火墙**: UDP 53535 端口需在防火墙开放
- **无认证**: 局域网内部发现协议默认不开启认证

### 3.4 可维护性要求

- **代码规范**: 遵循各语言官方编码规范
- **日志记录**: 关键操作需记录日志 (INFO/ERROR)
- **错误处理**: 优雅降级，网络异常不导致程序崩溃

## 4. 用户场景

### 场景 1: 服务注册与自动发现

**参与者**: AI 服务开发者 / AI 工具服务

**前置条件**: 
- 服务已部署并运行在局域网内
- 已创建 JSON 配置文件

**基本流程**:
1. 开发者创建服务配置文件 (service_config.json)
2. 启动 aiecho-agent 并指定配置文件
3. 服务代理自动在 UDP 53535 端口监听
4. 服务启动时自动广播公告消息
5. 客户端扫描时能发现该服务

**后置条件**:
- 服务已被客户端发现并可调用

---

### 场景 2: AI Agent 动态发现服务

**参与者**: AI Agent / AI Scanner 客户端

**前置条件**: 
- 局域网内已有注册的服务

**基本流程**:
1. AI Agent 调用 aiecho-scan 命令
2. 客户端发送 UDP 广播
3. 收集所有响应服务的基本信息
4. 并发请求各服务的 /ai_manifest 获取完整能力描述
5. 输出标准化的服务列表 (JSON)

**后置条件**:
- AI Agent 获得完整的服务能力列表，可动态选择调用

---

### 场景 3: 实时监听服务变化

**参与者**: 客户端 / 服务端

**前置条件**: 
- 客户端开启监听模式

**基本流程**:
1. 客户端启动监听模式 (aiecho-scan --watch)
2. 监听 UDP 53535 端口的公告消息
3. 新服务上线时收到 ANNOUNCE 消息
4. 服务下线时收到 GOODBYE 消息
5. 自动更新本地服务列表文件

**后置条件**:
- 服务列表文件实时反映局域网服务状态

---

### 场景 4: 服务调用

**参与者**: AI Agent / AI 服务

**前置条件**:
- AI Agent 已通过发现机制获取服务 Manifest

**基本流程**:
1. AI Agent 解析 Manifest 中的 capabilities
2. 根据 input_schema 构造请求参数
3. 向服务 base_url + invoke 端点发送请求
4. 接收响应并根据 output_schema 解析结果

**后置条件**:
- AI Agent 成功调用 AI 工具服务

### 核心组件

| 组件 | 说明 |
|------|------|
| aiecho-agent | 服务端代理，响应发现请求，提供 Manifest |

## 6. 术语表

| 术语 | 定义 |
|------|------|
| UDP 广播 | 向局域网广播地址 255.255.255.255 发送 UDP 数据包 |
| Manifest | 服务的完整能力描述 JSON，包含 meta、capabilities、endpoints 等 |
| Capability | 服务的具体能力/动作，如 "convert_pdf_to_md" |
| input_schema | 能力调用的输入参数 JSON Schema |
| output_schema | 能力调用的输出结果 JSON Schema |
| 服务公告 | 服务上线时主动广播的 ANNOUNCE 消息 |
| 服务告别 | 服务下线时广播的 GOODBYE 消息 |
